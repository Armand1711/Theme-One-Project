# Connect the Hidden Bonds - Project Context & Development Guide

## Project Overview
**Connect the Hidden Bonds** is an interactive heritage puzzle web application centered on the 1886 Union Masonic Temple in Kimberley. The core narrative is **discovering hidden social bonds** through four interconnected puzzles that reveal how immigrant communities and local networks sustained the temple and supported each other during economic challenges.

**Key Theme**: Users should feel they are *discovering mysteries* and *uncovering secrets*, not receiving direct instruction. Content is mysterious, evocative, and draws users in with intrigue.

---

## Project Goals
1. **Educational + Emotional**: Teach heritage history while creating an engaging discovery experience
2. **Linear Narrative**: One continuous story arc across four puzzles (Puzzle 1 → 2 → 3 → Final Reflection)
3. **Accessible & Responsive**: Works smoothly on desktop and mobile
4. **Premium Visual Experience**: Professional UI with advanced CSS (gradients, animations, backdrop filters)
5. **Mystery-Driven Engagement**: Content sounds mysterious rather than didactic

## Puzzle Mechanics

### Puzzle 1: Drag-and-Drop (Image Tiles)
- **HTML**: Draggable divs with `backgroundImage` and `backgroundPosition` split into quadrants
- **Interaction**: Drag from `.puzzle-source` to `.puzzle-target` slots
- **Validation**: Correct slot matches piece ID
- **Feedback**: Slot highlights on drag-over, turns green on correct placement
- **Animation**: GSAP placement animation, piece scaling on drag
- **Logic**: Solution object maps slot ID to piece ID; when all correct, trigger completion

### Puzzle 2: Symbol Match (Drag-Match)
- **HTML**: Symbol cards in source, meaning slots in target
- **Interaction**: Drag symbol from source to target slot
- **Validation**: Correct symbol-to-meaning pairing
- **Feedback**: Slot highlights, symbol scales on placement
- **Animation**: GSAP line draws + piece celebration animations
- **Logic**: Solution object maps slot ID to symbol ID; when all match, trigger completion

### Puzzle 3: Network Connection (Node Click)
- **HTML**: Positioned `.network-node` divs inside `.network-canvas`
- **SVG**: Lines drawn dynamically on canvas using mouse click positions
- **Interaction**: Click first node (selected state), click second node to connect, lines animate with GSAP
- **Validation**: Check if all required connections exist
- **Feedback**: Selected node highlights (green border/background), line animates on draw
- **Animation**: `strokeDasharray` animation for line drawing
- **Logic**: Correct connections array; when all found, trigger completion

## Responsive Design

### Breakpoints & Adjustments
- **Default**: Desktop (1050px+)
- **Tablet (860px–1050px)**:
  - Menu grid: 1 column
  - Hero height reduced
  - Stats flex-wrap
- **Mobile (<860px)**:
  - Puzzle layout: grid-template-columns 1fr (stacked)
  - Puzzle source/target: smaller min-heights
  - Font sizes reduced via clamp()
  - Pieces and slots: reduced heights
  - Buttons: full width with reduced padding

### Mobile-Friendly Patterns
- Use `clamp()` for responsive font sizing
- Avoid absolute positioning in responsive contexts
- Use media queries for grid/flex adjustments
- Touch-friendly tap targets (min 44px)

## Key Implementation Patterns

### Module Structure
Each screen is an `export function init<Screen>()` that:
1. Creates HTML string with template literals
2. Sets GSAP animation start states
3. Attaches event listeners
4. Handles callbacks to next screen

### State Management
- `completedPuzzles` tracked in `main.js`
- Progress updated after each puzzle completion
- Menu updates unlock state based on progress
- No external state library needed

### Animation Patterns
```javascript
// Set initial state
gsap.set('#element', { opacity: 0, y: 30 });

// Animate in
const tl = gsap.timeline();
tl.to('#element', { opacity: 1, y: 0, duration: 0.6, ease: 'power2.out' });
```

### Drag-and-Drop Pattern
```javascript
// dragstart: store data
e.dataTransfer.setData('text/plain', id.toString());
e.dataTransfer.effectAllowed = 'move';

// dragover: show drop zone
slot.classList.add('drag-over');
e.preventDefault();

// drop: place element
const dragged = e.dataTransfer.getData('text/plain');
// Move element to slot and validate
```

## Styling Best Practices

### Colors
- Use CSS variables for consistency
- Warm earth tones for heritage feel

- Use GSAP for complex sequences

## Common Tasks & Patterns

### Add New Puzzle
1. Create `js/puzzle<N>.js` with `export function initPuzzle<N>()`
2. Define interaction type (drag, click, etc.)
3. Wire into `main.js` flow
4. Add CSS styles to `css/puzzle1.css`
5. Update menu in `puzzleMenu.js`

### Modify Content
- Search for exact text and replace in JS files
- Keep tone mysterious and discovery-focused
- Avoid direct commands; use evocative language

### Debug Puzzles
- Check browser console for errors
- Verify event listeners are attached
- Confirm GSAP animations don't block interactions
- Test drag-and-drop with mouse and touch

### Responsive Fixes
- Add media query breakpoints as needed
- Test at 375px (mobile), 768px (tablet), 1024px (desktop)
